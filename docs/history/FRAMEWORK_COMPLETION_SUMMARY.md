## Architecture Highlights

### Master Orchestrator
- **Assesses** situation and desired outcome
- **Decomposes** complex tasks into iterations
- **Selects** appropriate agent roles
- **Monitors** execution in real-time
- **Validates** results against criteria
- **Applies** learnings from feedback

**Key Insight**: Orchestrator makes decisions but never executes work directly.

### Iteration Structure
Every iteration has:
- **CONTEXT.md**: Shared knowledge for agents
- **COMPLETION_CRITERIA.md**: Objective validation checklist
- **README.md**: Execution guide
- **agent-prompts/**: Generated prompts for each agent

### Orchestration Tools
- **generate_prompts.py**: Template ‚Üí Prompts (with token budget estimates)
- **orchestrate_full.sh**: Full automation (generate ‚Üí launch ‚Üí monitor ‚Üí validate)
- **monitor_enhanced.sh**: Intelligent progress tracking
- **validate_iteration.sh**: Deterministic validation
- **collect_feedback.sh**: Feedback synthesis and learning application
